Architecture

The system is a federation of Ruby, Swift, and SwiftUI components communicating over USB (gphoto2), WiFi (HTTP JSON-RPC), HTTP IPC, and WatchConnectivity. The camera source is abstracted behind a protocol so USB, WiFi, and generic AVFoundation cameras are interchangeable at runtime. The CameraSource protocol unifies all camera types: capture(), capturePreview(), adjustISO(), toggleRecording(). Similarly, LightController unifies Nanoleaf REST and HomeKit HMHomeManager.

Why WiFi, Not USB, for the App Store

The CLI uses gphoto2 over USB—fast, reliable, battle-tested. But macOS App Store apps live in a sandbox that blocks IOKit (USB access), killing ptpcamerad (requires sudo), and loading dynamic plugins from outside the app bundle. Sony's WiFi CCAPI (PC Remote over HTTP) is just REST calls. Only needs com.apple.security.network.client. Even Adobe Lightroom's Mac App Store version can't do USB tethering—we're in good company.

Body Pose

Full body pose detection via VNDetectHumanBodyPoseRequest. Currently recognizes arms raised (both wrists above both shoulders), which triggers a 3-second timer shot—enough time to lower your arms and pose. Body pose and hand gesture detection run simultaneously on the same AVCaptureSession video frames. The Vision framework handles the multiplexing.

Tongue Snap

An AVAudioEngine impulse detector: RMS threshold 0.12, duration gate <45ms (rejects sustained sounds like speech), 0.8-second cooldown. Buffer size 512 frames (~11.6ms at 44.1kHz), exponential moving average baseline (alpha 0.001). A spike that exceeds baseline + 0.12 and lasts less than 45ms fires the shutter.

Multi-Camera & Multi-Audio

Record from multiple cameras simultaneously. Each source gets its own preview window, which can be placed on any connected display via the Displays menu. The MultiCameraManager maintains independent CameraSlot instances with their own preview pollers and target screens. Camera sources are discovered via AVCaptureDevice.DiscoverySession with device types .builtInWideAngleCamera and .external. Sony cameras appear as external devices when connected via USB; via WiFi, they use the CCAPI HTTP source.

Audio recording supports any combination of available inputs as independent tracks. Each input gets its own AVAudioEngine writing to a separate WAV file, synchronized by timestamp. The MicrophoneMonitor displays real-time signal quality for all available microphones: Signal quality labels: Silent (<-60dB), Weak (-60 to -40dB), Good (-40 to -12dB), Strong (-12 to -3dB), Loud (-3 to 0dB), CLIPPING (0dB). Color-coded in the Microphone menu.

Save Destinations

Captured photos can be routed to multiple destinations:

  • File System — NSSavePanel with configurable default folder
  • Photos.app — PHPhotoLibrary with automatic album creation (DEAD/WALK/MAN 3000)
  • Camera SD Card — photo stays on camera, no download
  • File System + Photos.app — saves to both simultaneously

Sony Imaging Edge saves wherever it pleases and makes you hunt for the file. We ask politely, via the standard macOS save dialog.

Lighting

Two light control paths: direct Nanoleaf REST API for the Canvas E528 (17 panels, 43 effects), and Apple HomeKit for any compatible light in any Home.app home. The LightController protocol abstracts both: on(), off(), setBrightness(), setColor(h:s:b:). Switch between Nanoleaf and HomeKit in the Lights menu.

Camera Connection

USB Mode (gphoto2)

The Gantt chart rendered by Cockpit3000::Gantt in Tufte mode—ocean blues for done tasks, warm amber for WIP. Sony's USB implementation is, characteristically, hostile to the user. Read these instructions before connecting:

  1. Set USB Connection to PC Remote: MENU → Setup → USB Connection → PC Remote. Mass Storage and MTP will not work.
  2. Use a good cable. Not all USB-C cables carry data.
  3. Connect directly to your Mac's USB-C port. No hubs.
  4. Power cycle after changing settings. Off, change, on, plug in.
  5. macOS will steal the connection. ptpcamerad grabs the camera instantly. sonyctl kills it automatically.

WiFi Mode (Sony CCAPI)

For iOS, watchOS, and cable-free operation. HTTP JSON-RPC to port 8080. Sony cameras broadcast urn:schemas-sony-com:service:ScalarWebAPI:1 via SSDP. The camera IP is typically 192.168.122.1, shown on the camera screen when WiFi is active.

  1. Enable PC Remote over WiFi: MENU → Network → PC Remote → PC Remote Cnct Method → Wi-Fi
  2. Enable WiFi: MENU → Network → Wi-Fi → Wi-Fi Connect → ON
  3. Connect your device to the camera's WiFi: DIRECT-xxxx:ILCE-7C
  4. CLI: ruby ruby/sonyctl.rb --wifi 192.168.122.1 status
  5. macOS: ./sonyctl-live --wifi 192.168.122.1 or Camera menu → Sony WiFi

Generic Camera (AVFoundation)

Any camera that macOS recognizes works as an alternative to the Sony: built-in FaceTime camera, USB webcams, virtual cameras (OBS, Camo), external capture cards. Features are limited compared to Sony (no ISO/aperture control, basic capture only). In sonyctl-live, select from the Camera menu.
